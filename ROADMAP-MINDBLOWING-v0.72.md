## 1. Vision

cuneiform-mcp today is 110 tools that *retrieve and score* — co-occurrence statistics, trigram Jaccard, embedding cosines. The next arc turns it into an instrument that *discovers and predicts*: a self-critiquing classifier that mines its own mistakes for real intertexts, a graph engine that recruits its own validation labels without circularity, the field's first **neural** cuneiform model, and the first **public, DOI'd benchmark** the subfield can compete on. The through-line is that every one of these is buildable by one person from the *already-stacked* assets (36,498 sign streams, a 96,654-window chunk index, period metadata, the 47-pair validation store), and every one of them produces a sentence the JOHD Discussion Paper cannot currently write. We go from "a good retrieval toolkit" to "neural Assyriology with a published benchmark and a falsifiable discovery ledger."

## 2. THE FLAGSHIP — CuneiBERT

**A from-scratch masked-sign transformer: the first neural language model of cuneiform.**

### What it is
A small (~6-layer / 256-dim, ~5–8M param) BERT-style bidirectional masked-**language** model trained from scratch on the 36,498 ABZ-coded sign streams in `all-signs-full.json` (5.26M tokens). The vocabulary is only **2,082 signs** — small enough that the whole thing trains in hours in a single free Colab session. It ships as a new MCP tool `restore_lacuna_neural` that, given a sign stream with `X` damage positions, returns a *calibrated* softmax over candidate signs, shown side-by-side with the existing bigram+sign2vec restorer.

### Why it's a genuine first
There is no from-scratch sign-level masked-language model for cuneiform. The repo's headline today is "co-occurrence statistics." CuneiBERT changes it to "neural Assyriology with a head-to-head benchmark." Three properties make it *uniquely* solo-feasible here:
- The **2,082-sign vocab** makes a useful model tiny and trainable on consumer hardware.
- The corpus's `X` damage convention **is the native `[MASK]`** — the training objective is handed to us by the data format.
- The **frozen 500-gap restoration benchmark already exists** (`lacuna-bleu-calibration-samples.json`, seed `20260525`, baseline top1=0.182 / top5=0.36 / MRR=0.26), so the result is a clean apples-to-apples comparison, not a vibe.

It also unlocks an entire second wave — contextual per-occurrence sign embeddings (a drop-in for static sign2vec), perplexity-based anomaly hunting, witness-conditioned infilling — making it the *root* of a neural branch, not a one-off.

### The wow demo
A single table in the methods paper / README, on the **exact same frozen 500-gap set**:

| Restorer | top1 | top5 | MRR |
|---|---|---|---|
| bigram + sign2vec (baseline, pinned) | 0.182 | 0.36 | 0.26 |
| **CuneiBERT** | ? | **target ~0.6–0.7** | ? |

The credible target is roughly *doubling top5*, because the model sees the whole bidirectional line instead of one bigram of context. The screenshot artifact: `restore_lacuna_neural` on a real damaged tablet printing a calibrated ranked sign list next to the old restorer's, plus a reliability diagram showing the neural confidences are honest (ECE near baseline). **Crucially, it never touches the v1.0 G2 gate** (no model self-labeling of pairs) — pure paper upside, zero integrity risk.

### The build path (hard boundary at a weights file)
- **TRAIN (Python, offline, one-time):** `scripts/cuneibert/{prepare_corpus,train,evaluate}.py`. HuggingFace `BertForMaskedLM`, 6 layers / 256 hidden / 4 heads / max_len 128, 15% dynamic masking, ~10–20 epochs, T4-minutes/epoch. **The single biggest correctness trap:** real `X` (7.3% of tokens) maps to a `DAMAGE` special token and is *excluded from MLM loss targets* — training on it would be self-labeling and would silently inflate accuracy. Tablet-level (not position-level) holdout. Export weights to a compact `f32` binary (mirrors the existing `sign-vectors.f32` convention).
- **INFER (TypeScript, in-server, runtime):** `src/cuneibert.ts` — a ~300-line pure-TS forward pass (hand-rolled matmul over `Float32Array`). Loads the `.bin` once, lazy-cached like `signInference.ts`. **The server keeps its 2-dependency footprint and stays `npx`-installable with no Python at runtime.** Register `restore_lacuna_neural` by copying the `restore_lacuna_semantic` block (`src/index.ts:5978`); route the softmax through the existing `recalibrateLacunaScores` Platt path. Write benchmark output to a *new* file so the 0.182/0.36 baseline stays pinned.

### Why it's solo-achievable
The data is verified ready, the benchmark harness already exists, and the model is deliberately tiny. Realistic effort: **2–3 weeks part-time** (can slip to 4 if TS↔PyTorch numeric parity fights back). The genuine time sink is the bit-exact TS port (layernorm eps, gelu variant, softmax order — needs a logit-parity test). **First step (de-risk the data, not the model):** a one-shot Python script that (a) confirms every non-X/non-numeric token maps to a vocab ID (report unmapped rate — high = tokenizer misaligned = plan stalls), (b) regenerates the exact 500 frozen gaps from `benchmark-lacuna-bleu.mjs`, and (c) reproduces the published top1≈0.182 with a trivial bidirectional bigram in your own harness. Reproducing 0.182 yourself is the gate that proves the comparison will be valid.

> **Toolchain note:** `torch` is not currently installed and the box runs Python 3.14.5 (torch wheels likely unavailable). Use a venv pinned to 3.11/3.12, or just train in Colab.

## 4. How they compose

This is an arc, not a grab-bag, and it has two spines. **The G2 spine** is the cheapest and highest-ROI: `find_transitive_sibling_candidates` (topology), `surface_genre_conflicts` (the model's corroborated mistakes), and the **Join Oracle** (external eBL adjudication) are three *independent, non-circular* generators of candidate labels. They share a back-end — all three feed the existing `record_validation_resolution` intake — so the natural wrapper is a single cron "night-shift" harness that runs all three and queues a worklist, rather than three separate agents. Crucially the Join Oracle is the *only* one whose confirmations are clean enough to count directly toward the ≥100-positive gate (eBL, not the model, decides), while the other two are human-adjudication generators that remove search cost but not judgment cost. **CuneiPair-100 is the publication multiplier on this spine:** every operator-confirmed pair the active-learning loop adds bumps the dataset version and the Zenodo deposit, so the private G2 grind becomes a public, citeable, PR-able leaderboard.

**The neural/discovery spine** is led by CuneiBERT, which is deliberately *orthogonal* to G2 (no pair self-labeling) so it carries zero integrity risk while opening the contextual-embedding / perplexity-anomaly second wave. The graph-and-chunk discovery tools cluster around one shared substrate — the chunk index inverted to a weighted tablet graph — so `detect_contamination_communities`, `discover_orphan_chains`, and the Cross-Language Bridge are siblings, not rivals; ship **one** explorer over that substrate, not three. GlyphPrint sits above everything as a multimodal QA layer and is the one strategic bet that builds genuinely new infrastructure (the first vision pipe), which is why it earns its own kill-switch gate. The Direction Inquisitor is the keystone that, once built, retroactively unblocks the deferred Transmission Time-Machine.

## 6. What we are deliberately NOT doing yet (and why)

- **No geographic Scriptorium Atlas.** `provenance` is **null for all 36,316 tablets** — a signature find-spot map is literally un-renderable without new enrichment. *Period* is present and rich, so diachronic work (Cross-Language Bridge) is fine; geography is not.
- **No "transmission river" demo yet.** The latest calibrated quotation network is **4 nodes / 5 edges with no direction field**. The river would be a near-empty trickle. It is *deferred behind* the Quotation-Direction Inquisitor, which must first create the directed graph.
- **No second/third discovery explorer.** Contamination network, orphan chains, and a "corpus constellation" all reuse the same anomaly/chunk-graph substrate. Shipping three is duplicated effort for one signal. Ship one.
- **No LLR-sign-crop paleography (GlyphPrint v2).** eBL `/annotations` returns `[]` — no bounding boxes exist. v1 is whole-tablet/grid embeddings only; the crop story is deferred until boxes materialize.
- **No RESTORE-cdli leaderboard alongside CuneiPair-100 (for now).** Near-duplicate "first FAIR benchmark" play; CuneiPair ties to the v1.0 model and wins if doing only one. Keep only if bandwidth allows — they share the Zenodo harness.
- **No auto-labeling, anywhere, ever.** Every candidate generator (transitive, genre-conflict, orphan-chain, contamination) emits *hypotheses for human verdict*. A → B and B → C sibling does **not** force A → C (the BM.77056↔K.5896 curriculum-vs-centerpiece case already proves false transitivity). Auto-accepting reintroduces the circularity the methods paper forbids. Only **eBL-adjudicated** Join Oracle hits and human-confirmed verdicts count toward G2.
- **No tablet-image pipeline beyond GlyphPrint's gated probe.** Vision is genuinely new infrastructure with a real scientific failure mode (whole-tablet embeddings conflate shape/lighting/breakage with "the hand"). It gets exactly one kill-switch experiment before any multi-week commitment.
