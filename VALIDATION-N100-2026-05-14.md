# N=100 trigram re-validation — confidence interval tightened
*Run date: 2026-05-14. Closes polish-queue P4 "Run a clean N=100 trigram
validation when eBL is healthy."*

## Why this run exists

The 2026-05-14 morning stress test (seed=137, N=100 target, recorded in
`trigram-validation-N100-seed137.json` v0) was truncated at N=26 by an
eBL outage — HTTP started returning errors at fetch=550 and continued to
the max-fetch cap. The truncated run reported recall@15 = 11/57 = 19.3%,
but with only 57 known siblings the CI was wide enough that 19.3% was
not a reliable comparator to the 25.3% baseline (N=50, seed=42, 87
siblings).

This run completes the seed=137 sweep on a healthy eBL.

## Method

- `scripts/validate-matcher-trigram.mjs 100 137 15` — seed=137 (same as
  truncated run), N=100 target keepers, recall@15.
- Tokenizer: matches `src/signsIndex.ts` as shipped 2026-05-14 (drops
  trigrams with ≥2 X tokens — commit `6d79d5a`,
  `X-FILTER-EXPERIMENT-2026-05-14.md`). Recall@15 is filter-invariant on
  the baseline but the median/mean rank numbers reflect live behavior.
- Phase 1: HTTP fetches `/api/fragments/<id>` until 100 keepers found
  (keeper = target has ≥1 join sibling that's also in the trigram corpus).
- Phase 2: scores each keeper against the full 35,330-fragment trigram
  index, finds rank of each known sibling.

The 35.6% zero-trigram-overlap floor (64/180) holds firm — same as the
seed=42 baseline (31/87 = 35.6%). This is the hard algorithmic limit:
siblings with zero shared trigrams cannot be ranked by Jaccard.

## Comparison to prior runs

| run | seed | sample | recall@15 | 95% CI (approx) |
|---|---|---|---|---|
| baseline       | 42  |  87 siblings | 25.3% | [17%, 35%] |
| truncated      | 137 |  57 siblings | 19.3% | [11%, 30%] |
| **full N=101** | 137 | 180 siblings | **21.1%** | [16%, 27%] |
| **combined**   | both | **267 siblings** | **22.5%** | **[17%, 28%]** |

The original baseline (25.3%) sat on the optimistic end. The full
seed=137 result (21.1%) sits on the more typical end. The combined
estimate of 22.5% across 267 siblings is our tightest data point, with
the 95% CI roughly [17%, 28%].

The truncated run's 19.3% on N=57 was within sampling noise of the full
21.1%. The eBL outage didn't materially change the result; it only
shrunk the sample.

## Algorithmic interpretation

Recall@15 ≈ 22% means the live `find_parallel_text` tool surfaces the
correct known sibling in its top-15 results about one query in five.
That's substantially above the 3.4% baseline of eBL's published
lineToVec-only matcher and is the published number we should use going
forward.

The 35% zero-overlap floor caps any improvement attainable by the
trigram signal alone. To meaningfully break above ~25%, we'd need to
add a second signal — paleography, image similarity, or composition-aware
retrieval. None of those are hobby-scale.
