# cuneiform-mcp v0.16 — Anomaly Surface Spec

*Status: DRAFT, 2026-05-16.  Author: Dane + Claude.  Blocked-by: v0.15 (semantic embeddings) shipping.*

---

## Discovery thesis

The corpus-viz v0.1 build produced **19,787 tablets in the trigram graph** and **17,486 connected components** — meaning ~88% of long-enough tablets have fewer than 2 lexical siblings. That huge tail is the **discovery surface**.

Most singletons are dull (short administrative tablets, idiosyncratic vocabularies). The interesting subset is:

> Long-enough tablets, in well-attested genres, that have **no neighbors in either the lexical (trigram) or the thematic (embedding) graph** — *bi-orphans*. These are the candidates for previously-unknown compositions, miscatalogued fragments, or rare witnesses of poorly-attested texts.

v0.16 ships the MCP tools that surface this set.

## Tool 1 — `find_anomalous_tablets`

### Purpose

Return a ranked list of tablets matching one of several anomaly criteria. Replaces the "manually browse corpus-viz looking for outliers" workflow with a queryable API.

### Inputs

```ts
{
  anomaly_type: "bi_orphan" | "lexical_singleton" | "thematic_orphan" |
                "cluster_genre_misfit" | "cluster_period_misfit" |
                "low_lexical_high_thematic" | "low_thematic_high_lexical";
  min_sign_count: number;     // require ≥ N signs to filter out short fragments; default 100
  period_filter?: string;     // optional period
  genre_filter?: string;      // optional genre
  max_results: number;        // default 20, max 100
}
```

### Anomaly types

| Type | Definition | Why it matters |
|------|------------|----------------|
| `bi_orphan` | No lexical neighbors above min-jaccard AND no thematic neighbors above min-cosine | Highest-priority discovery candidates |
| `lexical_singleton` | Singleton in corpus-viz graph (no neighbors at min_jaccard ≥ 0.30) | Either rare witness or short/idiosyncratic |
| `thematic_orphan` | No embedding neighbors above min_cosine ≥ 0.5 | Unique distributional profile — possibly novel topic |
| `cluster_genre_misfit` | Tablet's metadata genre ≠ the dominant genre of its embedding cluster | Possibly miscatalogued or genre-bridging |
| `cluster_period_misfit` | Tablet's metadata period ≠ the dominant period of its embedding cluster | Possibly misdated; useful for redating studies |
| `low_lexical_high_thematic` | Few trigram neighbors but many embedding neighbors | "Different vocabulary, same topic" — paraphrase candidates, lexical-list parallels |
| `low_thematic_high_lexical` | Many trigram neighbors but few embedding neighbors | "Same vocabulary, different topic" — formulaic-text outliers, colophon-template artifacts (cf. v0.14.4) |

### Outputs

```ts
{
  query: {...},
  anomaly_count: number,
  anomalies: Array<{
    tablet_id: string,
    anomaly_type: string,
    scores: {
      lexical_neighbor_count: number,
      lexical_max_jaccard: number,
      thematic_neighbor_count: number,
      thematic_max_cosine: number,
      cluster_genre_consistency?: number,  // 0-1
      cluster_period_consistency?: number, // 0-1
    },
    metadata: {
      period?: string,
      genre?: string,
      city?: string,
      designation?: string,
      sign_count: number,
    },
    interpretation: string,  // human-readable, includes [my synthesis] marker
    follow_up: string,       // suggested next step for an Assyriologist
    ebl_url: string,         // deep link to ebl.lmu.de
  }>,
  warnings: string[],
  index_stats: {
    total_tablets_in_lexical_graph: number,
    total_tablets_in_embedding_index: number,
    total_intersect: number,  // tablets present in BOTH (the true anomaly-detection surface)
  },
}
```

### Implementation notes

1. Load 3 sources at startup (all lazy + cached):
   - `~/Desktop/corpus-viz/graph.json` — lexical neighbors per tablet
   - `~/.cache/cuneiform-mcp/tablet-neighbors.json` — thematic neighbors per tablet (v0.15)
   - `data/tabletMetadata.json` — period/genre/city/designation
2. Precompute for each tablet:
   - `lex_count` = number of neighbors in graph.json with score ≥ min_jaccard (built into graph.json already)
   - `them_count` = number of neighbors in tablet-neighbors.json with score ≥ min_cosine
   - `cluster_id_lex` = component from corpus-viz
   - `cluster_genre_consistency` = (# tablets in cluster with same metadata genre as this tablet) / cluster_size
   - `cluster_period_consistency` = same for period
3. For each anomaly_type, apply the corresponding filter + rank by relevant score
4. Build the human-readable `interpretation` string with the synthesis flag

### Validation plan

After implementation, manually inspect the top-20 `bi_orphan` results:
- Check eBL for catalog notes
- Look for partial publication history in JNES/JCS/OrAnt
- Classify each: (a) genuinely-unique composition, (b) miscatalogued fragment, (c) very-short tablet false-positive, (d) signs-cache parsing error

Target outcome: at least 5 of 20 candidates should be class (a) or (b). If fewer, tune the methodology (raise min_sign_count, add length-normalization).

## Tool 2 — `describe_anomaly`

### Purpose

Given a specific tablet, return a structured anomaly report. Used by an Assyriologist who's curious about why a particular tablet was flagged.

### Inputs

```ts
{
  tablet_id: string;
}
```

### Outputs

```ts
{
  tablet_id: string,
  exists_in_lexical_graph: boolean,
  exists_in_embedding_index: boolean,
  metadata: { ... },
  lexical: {
    neighbor_count: number,
    top_neighbors: [{ id, jaccard }, ...],  // top 5
    component_id?: number,
    component_size?: number,
    component_dominant_genre?: string,
  },
  thematic: {
    neighbor_count: number,
    top_neighbors: [{ id, cosine, period?, genre? }, ...],  // top 5
  },
  anomaly_scores: {
    is_bi_orphan: boolean,
    is_lexical_singleton: boolean,
    is_thematic_orphan: boolean,
    is_genre_misfit: boolean,
    is_period_misfit: boolean,
  },
  reasons: string[],     // human-readable explanations
  follow_up: string[],   // ordered next steps
}
```

## Tool 3 — `discovery_surface_stats`

### Purpose

Top-level stats on the discovery surface itself. Useful for tracking how the surface changes as exclusions + thresholds are tuned.

### Inputs

None.

### Outputs

```ts
{
  total_tablets_in_lexical_graph: number,
  total_tablets_in_embedding_index: number,
  total_intersect: number,
  bi_orphan_count: number,
  lexical_singleton_count: number,
  thematic_orphan_count: number,
  // Length-bucketed counts for the >= 100 signs subset
  by_length: {
    "100-300": { bi_orphans: number, total: number },
    "300-1000": { bi_orphans: number, total: number },
    "1000+": { bi_orphans: number, total: number },
  },
  // Per-genre breakdown
  by_genre: Record<string, { bi_orphans: number, total: number }>,
}
```

## Discovery-yield projection

Based on the corpus-viz v0.1 numbers:
- 19,787 tablets in lexical graph, 17,486 components → ~9,500 are absolute lexical singletons
- v0.15 embeddings index covers ~28,665 tablets (more than the lexical graph since min sign count differs)
- The **intersect** is what matters: roughly 15,000–18,000 tablets in BOTH indices
- Applying `min_sign_count ≥ 100` knocks out short fragments → estimated ~3,000–5,000 tablets clear the bar
- Of those, `bi_orphans` are probably 100–500 tablets

**The expected real discovery yield** from manual inspection of the top-200 bi-orphans is probably 10–40 tablets that are genuinely interesting (unique compositions, miscatalogued fragments, or poorly-attested-text witnesses). That's **a publishable Assyriology result** — the methodology + the discovery list.

## Open questions

1. **Length normalization** — embedding magnitude is dominated by long tablets. Should we use length-normalized cosine? *(Test empirically on v0.15 output.)*
2. **Genre-imbalance bias** — administrative tablets are massively overrepresented. Should anomaly detection condition on genre? *(Probably yes — rank within-genre rather than globally.)*
3. **False-positive class from v0.14.4** — colophon-template prototypes were already excluded. Are there other systematic FP classes we'll surface? *(Have to validate empirically; reserve a follow-up exclusion pass.)*
4. **What to do with bi-orphans that have NO metadata?** — many tablets in the eBL corpus don't have period/genre. They're the hardest to interpret but also the most under-studied. Should the tool emphasize or de-emphasize them?
