## Algorithm details

```ts
// For each candidate, compute contiguous-run of matching trigrams in
// the target's ordered position stream:
let longestRun = 0, currentRun = 0;
for (const tri of targetTrigramsOrdered) {
  if (candSet.has(tri)) {
    currentRun++;
    if (currentRun > longestRun) longestRun = currentRun;
  } else {
    currentRun = 0;
  }
}
// Same run-bonus formula as v0.18.2 fuzzy_parallels:
const runFactor = Math.min(1, longestRun / Math.sqrt(targetTrigrams.size));
const runBonus = 0.5 * runFactor;  // capped at +50% lift
const finalScore = jaccard * (1 + runBonus);
```

Sort by `final_score` instead of bare `jaccard`. Output rendering shows all four values (`final`, `jaccard`, `run`, `bonus`) for full auditability.
