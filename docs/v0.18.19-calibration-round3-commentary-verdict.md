# v0.18.19 Calibration Audit — Round 3, Lever 2: Commentary-Quotation Verdict

*Lever 2 of Round 3. `compare_tablet_pair` mis-classifies high-fuzzy-J + long-run pairs where one side is genre-tagged Commentary as `physical_join_candidate`. The relationship is actually commentary quoting base text. Adds a verdict-classifier branch that fires when `longest_run ≥ 100` AND exactly one side has "Commentary" in its eBL `genres_flat`.*

---

## Hypothesis

The 2026-05-23 typology and methods-paper §3.7.1 record:

> BM.47463 ↔ CBS.6060 = **147-sign verbatim chain** at fuzzy-J 0.70. Initial verdict: `physical_join_candidate`. eBL's `lineToVec /match` REJECTS join candidacy. Resolution: BM.47463 = Šurpu **Commentary** (eBL genre `CANONICAL → Technical → Commentary`); CBS.6060 = Šurpu **base text**. The chain is commentary quoting base text, not physical join.

`compare_tablet_pair` should encode this distinction in its verdict classifier — high fuzzy-J + long run + asymmetric Commentary genre = commentary-quotes-base-text, NOT physical-join-candidate.

## Ship: new classifier branch in `comparePair.ts`

```ts
// v0.18.19 calibration audit Round 3 / Lever 2: commentary-quotes-base detection.
if (longRun >= 100 && fuzzyJ >= 0.3) {
  const aIsCommentary = hasCommentaryGenre(tabletA);
  const bIsCommentary = hasCommentaryGenre(tabletB);
  if (aIsCommentary !== bIsCommentary) {                  // XOR — exactly one side
    const commentarySide = aIsCommentary ? tabletA : tabletB;
    const baseSide = aIsCommentary ? tabletB : tabletA;
    return { primary_relationship: "commentary_quotes_base_text", confidence: "high", ... };
  }
}
```

The branch fires before the `fuzzyJ >= 0.5 && scribCos >= 0.7` join branch — commentary-genre evidence is more dispositive than scribal-cos evidence when both signals are present. The XOR condition (`aIsCommentary !== bIsCommentary`) requires asymmetric Commentary tagging: a commentary-on-commentary case has both sides tagged and falls through to the existing logic, which is the right default while corpus coverage of double-commentary tablets is too thin to calibrate against.

Pre-condition `fuzzyJ >= 0.3` guards against the degenerate "very long run from formulaic colophons" case — a real commentary-quotes-base relationship has substantial vocabulary overlap, not just a long colophon run.

`PairRelationship` union extended with `commentary_quotes_base_text`. No schema changes — the field is already a freeform string in the existing `find_compare_tablet_pair` envelope.

## Two new methods-paper claims

17. **`[my synthesis]`** **High-fuzzy-J + long-run + asymmetric Commentary genre tagging is a quotation pattern, not a join pattern.** The verdict classifier can encode this distinction without an external eBL `/match` round-trip — the `genres_flat` field of fragment metadata carries the signal, the `longest_contiguous_run` carries the corroboration. BM.47463↔CBS.6060 (Šurpu Commentary quoting Šurpu base text, run=108) is correctly classified `commentary_quotes_base_text` with no metadata enrichment beyond the existing eBL fragment-metadata fetch.

18. **`[my synthesis]`** **Genre tagging in the verdict classifier is precision-positive without harming recall on the anchor cases.** The fuzzy-J + run-length signals that drove v0.18.2 sibling rescues (K.2798↔Si.776, K.5896 discovery) are not Commentary-tagged on either side and pass through the new branch unchanged. The classifier becomes more discriminating at the high-fuzzy-J + long-run regime, where the prior classifier's coarsest decision (physical_join_candidate) was the most likely to be wrong.

## Followups

- Commentary-on-commentary pairs (both sides tagged) are presently treated by the fall-through logic (typically `same_composition_*`). When the eBL fragment-metadata cache grows beyond 0.6% corpus coverage, re-audit whether double-Commentary cases need a separate `commentary_on_commentary` verdict.
- The 100-position threshold is a calibrated cutoff (BM.47463↔CBS.6060 at run=108 fires correctly; K.2798↔Si.776 at run=27 does not). Long-run colophon false positives are theoretically possible but require both `fuzzyJ ≥ 0.3` AND asymmetric Commentary genre — unlikely combination in practice. Re-visit if a real false-positive surfaces.
