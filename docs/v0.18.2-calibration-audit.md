# v0.18.2 Calibration Audit Findings — 2026-05-16

*Following the v0.18.0 → v0.18.1 lacuna-restorer calibration that lifted top-1 precision 23% → 92% via a single-line length-factor change, I audited the remaining ranking heuristics across the toolchain. Three concrete fixes shipped.*

---

## Audit findings

### 1. Bi-orphan thematic threshold (0.60) was capturing genuine sibling pairs

The K.2798 ↔ Si.776 pair (CONFIRMED Bīt salāʾ mê manuscript siblings via v0.17 fuzzy_J=0.41) scores at thematic cosine **0.5587**. This is below the v0.16 0.60 bi-orphan threshold — so this confirmed-sibling pair was being classified as "thematic orphan."

This was a systematic mis-calibration: real manuscript siblings with mild distributional divergence (different scribes, short tablets, partial preservation) score in the 0.50–0.59 cosine band, but the threshold was excluding them as anomalous.

**Distribution analysis:**

| Thematic threshold | Corpus-wide bi-orphan count |
|---|---|
| 0.60 (v0.16) | 167 |
| **0.50 (v0.18.2)** | **11** |
| 0.40 | 0 |

The 0.60 → 0.50 tightening drops bi-orphan count from 167 to 11 — a 93% reduction. The 11 remaining are the genuinely-isolated tablets (matching the v0.17 still-isolated list exactly).

**Fix:** lowered threshold to 0.50 in `anomalySurface.ts` bi_orphan + thematic_orphan classification.

### 2. Bi-orphan scoring formula was dominated by sign_count (~80%)

The v0.17 scoring was:
```
score = (sign_count - them_max_cos × 100) × x_ratio_penalty
```

Empirical decomposition on top-15 candidates showed `sign_count` contributed an average of **77.7%** of the raw score — the "isolation strength" signal (`them_max_cos`) was a minor modifier. Long tablets with moderate isolation were ranking above short tablets with strong isolation, even though the latter are stronger discovery candidates.

**Fix:** rebalanced to:
```
score = isolation_strength × sqrt(sign_count) × x_ratio_penalty
where isolation_strength = 1 - them_max_cos
```

Both axes now contribute meaningfully. `sqrt(sign_count)` ensures length still matters but doesn't dominate.

### 3. Fuzzy parallels treated contiguous runs and scattered noise identically

For the K.2798 ↔ Si.776 pair (fuzzy_J=0.41, 129 fuzzy matches), trigram-position analysis revealed:

```
K.2798 → Si.776 fuzzy match positions:
  Longest contiguous run: 29 positions (50-78)
  Top runs: 29, 21, 15, 11, 9
```

The 29-position contiguous run is strong text-section sibling evidence. The previous methodology scored only by total fuzzy_intersect / size — treating a 29-position run identically to 29 scattered matches.

**Fix:** added per-pair `longest_contiguous_run` tracking to `fuzzyIntersection()` + a run-bonus multiplier to the final score:

```
run_factor = min(1, longest_run / sqrt(query_trigrams))
run_bonus = 0.5 × run_factor              # capped at +50% lift
final_score = fuzzy_jaccard × (1 + run_bonus)
```

Ranking now uses `final_score` instead of bare `fuzzy_jaccard`.

## Calibration audit methodology (for future tooling)

For each tool's scoring heuristic, audit by:

1. **Decompose**: identify the formula's component terms
2. **Quantify**: on a sample of real outputs, what % of the score does each term contribute?
3. **Test edge cases**: do known-correct positives rank above known false-positives? At what threshold?
4. **Test with synthetic ground truth**: when you control the input (synthetic gaps, planted siblings), what's the precision/recall?
5. **Ship fixes empirically**: a one-line change that lifts a metric by 60+ points (v0.18.1) is more valuable than a sophisticated algorithm

The v0.18.1 + v0.18.2 calibration train collectively demonstrates that **methodological precision in cuneiform-discovery tooling is often calibration-limited rather than signal-limited** — a general principle worth applying to other corpora.
