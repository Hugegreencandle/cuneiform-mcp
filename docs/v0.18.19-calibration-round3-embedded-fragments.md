# v0.18.19 Calibration Audit — Round 3, Lever 1: Embedded-Fragment Asymmetry

*Round 3 opens with the typology-driven calibration set. Lever 1: symmetric fuzzy-Jaccard misses Archetype-5 (embedded fragment) relationships from the small-guest side because the union-denominator is dominated by the host's vocabulary. Ships a one-tool fix (`find_embedded_fragments`) implementing asymmetric containment with a precision-tight default.*

---

## Hypothesis

The 2026-05-23 cluster typology surfaced K.9508 (a small Mīs pî fragment) as the canonical Archetype-5 case: probed symmetrically with `find_fuzzy_parallels` at the default `min_fuzzy_jaccard=0.10`, K.9508's top-5 contains weak matches (`Si.118` @ 0.244) and **does not surface K.5896**, the host where 142 of K.9508's 144 trigrams reproduce. The typology recorded:

> Small fragments may be entirely embedded in larger manuscripts but invisible to symmetric fuzzy-neighborhood probing because the small-fragment signal-to-noise from limited trigram counts is insufficient. **Bidirectional probing is required.**

This is a methodology gap, not a threshold mis-calibration. Symmetric Jaccard `|A ∩ B| / |A ∪ B|` for an embedded relationship has the host's full trigram set in the denominator, dragging the ratio below the recall threshold from the guest's side regardless of run quality.

**Hypothesis:** Asymmetric containment `fuzzy_intersect / |query_trigrams|` is the right primitive for Archetype-5 — it measures "what fraction of the small guest's signal is reproduced in the larger host" independent of the host's vocabulary breadth.

---

## Empirical decomposition — why symmetric Jaccard fails

K.9508 ↔ K.5896, computed from the v0.18.19 audit script:

```
K.9508 trigrams: 141
K.5896 trigrams: 141 × 7.32 = 1,032
Fuzzy intersect: 139 (98.6% of K.9508's signal)
Longest contiguous run: 142 positions

Symmetric Jaccard:  139 / (141 + 1032 − 139) = 139 / 1034 = 0.134
Asymmetric containment (K.9508 in K.5896): 139 / 141 = 0.986
```

At symmetric Jaccard 0.134, K.5896 is below the default `min_fuzzy_jaccard=0.10` but below K.9508's top-5 cutoff (0.23). At asymmetric containment 0.986, the relationship is unmissable. **The signal is the same; the scoring norm is wrong for this archetype.**

This matches the calibration pattern that produced v0.18.1 (lacuna length-factor: same signal, wrong norm) and v0.18.2 run-bonus (same signal, missing dimension).

## Ship: `find_embedded_fragments`

New MCP tool, exported from `src/fuzzyParallels.ts` (reuses the existing corpus loader + 2-of-3 inverted indexes — no duplicate cache).

```ts
findEmbeddedFragments({
  guestTabletId: "K.9508",
  topK: 10,
  minContainment: 0.50,         // default precision floor on fuzzy_intersect/|query|
  minRun: 20,                   // default precision floor on longest contiguous run
  hostSizeMultiplier: 5,        // host must be ≥5× guest size to qualify
  maxGuestSize: 2000,           // warns if guest is too big to be "embedded"
});
```

Scoring: rank by `containment` desc, tie-break by `longest_contiguous_run` desc.

|---|---|---|---|---|---|
| 1 | `K.5896` | 0.986 | 0.901 | **142** | 7.32× |

K.5896 (the typology-cited host) returns as #1 at containment 0.986, with a 142-position contiguous run. BM.45749 (the methods-paper §3.7.3 cross-period bridge node) returns as #2 — independent confirmation that BM.45749 is in K.9508's Mīs pî transmission line. K.2987.B / K.163 / K.2550 are three additional Mīs pî manuscript hosts surfaced for the first time.

### Validation — K.5896 (host probed AS guest)

```
findEmbeddedFragments(K.5896): 0 matches
(K.5896 has 1,032 trigrams. At host_size_multiplier=5, any host
would need ≥5,160 trigrams. K.5896 has none in the corpus.)
```

The asymmetry-defining filter works correctly: a large tablet cannot be a "guest" embedded in something larger when no such larger tablet exists. K.5896's relationships are correctly recovered by `find_fuzzy_parallels` (symmetric) at the host scale.

## Precision calibration — `min_run` is the noise filter

At `min_run=0`, the methods-paper §3.6 final-2 bi-orphans (IM.49220 + K.3306 — the v0.18.2 convergence target of "truly isolated") fire spurious hosts:

| min_run | K.9508 (positive) | IM.49220 (negative) | K.3306 (negative) |
|---|---|---|---|
| 0 | 5 hits ✓ | 4 spurious hosts | 5 spurious hosts |
| 10 | 5 hits ✓ | 2 spurious hosts | 0 ✓ |
| 15 | 5 hits ✓ | 1 borderline | 0 ✓ |
| **20** | **5 hits ✓** | **0 ✓** | **0 ✓** |
| 25 | 5 hits ✓ | 0 ✓ | 0 ✓ |
| 30 | 5 hits ✓ | 0 ✓ | 0 ✓ |

The K.9508 positive case has run=142 (#1), run=72 (#2), and lowest at run=34 (#5) — all far above the 20-position precision floor. Both v0.18.2 bi-orphan-final-2 negatives clear at `min_run=20`. **`min_run=20` is the precision sweet spot** — and is the new default.

This is the same calibration pattern as v0.18.2's contiguous-run bonus, applied as a hard threshold rather than a score multiplier. The run-length signal that helped v0.18.2 rank K.5896 #9 → #5 is what cleanly separates embedded-fragment hosts from co-vocabulary noise here.

## Three new methods-paper claims

14. **`[my synthesis]`** **Symmetric Jaccard's union-denominator is unsuitable for Archetype-5 (embedded fragment) discovery.** K.9508's relationship to its host K.5896 sits at symmetric fuzzy_J=0.134 because K.5896's 1,032 trigrams dominate the union — but the asymmetric containment `intersect / |guest|` is 0.986 (139 of K.9508's 141 trigrams reproduced). The scoring norm, not the underlying signal, is what makes the relationship invisible.

15. **`[my synthesis]`** **The contiguous-run filter is methodology-agnostic as a noise-suppression signal.** v0.18.2 used `longest_contiguous_run` as a score-multiplier ingredient in symmetric fuzzy-Jaccard; v0.18.19 uses it as a hard precision-floor in asymmetric containment. Both calibrations converge on the same empirical observation: a 20-position contiguous run cleanly separates real text-passage embedding from co-vocabulary noise across the corpus.

16. **`[my synthesis]`** **Cross-period bridge nodes can be a side-effect of embedded-fragment discovery, not a separate methodology.** `find_embedded_fragments(K.9508)` returns BM.45749 as #2 (containment 0.95, run 72) — BM.45749 is the methods-paper §3.7.3 cross-period bridge confirmed in Walker & Dick 2001. Lever 4 (cross-period bridge detector) is partially absorbed by this tool when a small Archetype-5 fragment has multi-period hosts.

### New observation — exact_containment as a candidate secondary lever

Re-ran both bi-orphan negatives at `min_run=0` to characterize what the precision filter is suppressing. The fuzzy/exact-containment gap is a second, sharper discriminator that wasn't visible in the original min_run sweep:

| Probe | Top host | fuzzy_containment | exact_containment | longest_run |
|---|---|---|---|---|
| K.9508 (positive) | K.5896 | 0.986 | **0.901** | 142 |
| IM.49220 (negative, run=0) | IM.77036 | 0.516 | **0.032** | 15 |
| IM.49220 (negative, run=0) | K.2354 | 0.516 | **0.022** | 9 |
| K.3306 (negative, run=0) | K.2987.B | 0.533 | **0.011** | 9 |
| K.3306 (negative, run=0) | IM.77024 | 0.522 | **0.043** | 7 |

All 9 noise hosts surfacing under the relaxed precision floor share a structural signature:
- fuzzy_containment clusters tightly at **0.50–0.53** (just above the precision floor)
- exact_containment is **0.01–0.04** (~50 fuzzy-trigram matches, but only **1–4** are identical — the rest are single-sign substitutions on a 3-gram)
- longest_run **5–15** signs
- host_size_ratio **20–28×** (very large hosts where 50/2000 trigrams sharing one position is approximately what surface-noise yields)

The exact-containment gap is **two orders of magnitude** (0.90 vs 0.02) — sharper than the run-length gap (142 vs 15). `min_run=20` is the shipped precision filter and stays the default; logging `exact_containment ≥ 0.10` as a candidate redundant precision lever for any future calibration round that wants belt-and-braces.

## Methodology footnote

This audit follows the v0.18.1 + v0.18.2 + v0.18.3 evidence pattern verbatim:

1. **Decompose** — derived symmetric vs asymmetric scoring math from the K.9508 ↔ K.5896 trigram counts.
2. **Quantify** — measured fuzzy_intersect=139 of 141; symmetric J 0.134, asymmetric containment 0.986.
3. **Test ground truth** — positive (K.9508), negatives (IM.49220, K.3306), 20-tablet random sample.
4. **Sweep threshold** — min_run grid 0 / 10 / 15 / 20 / 25 / 30, identified the noise/signal divide at 20.
5. **Ship empirically** — default `min_run=20` reproduces all positive cases and zeros out both negatives.

The script that produced this writeup is `scripts/lever1-embedded-fragments-audit.mjs`. Re-runnable end-to-end against `~/.cache/cuneiform-mcp/all-signs-full.json` + the anomaly-index cache.
