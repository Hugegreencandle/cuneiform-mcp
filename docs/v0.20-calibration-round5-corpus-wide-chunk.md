# v0.20.0 Round-5 Calibration Audit — Corpus-Wide Chunk Discovery

Round-5 audits the v0.20.0 corpus-wide chunk-discovery surface: the chunk-hash index, `find_formulaic_passages`, `trace_chunk_diffusion`, and `build_citation_graph`. Unlike Rounds 1-4, there is no 20-tablet random sample — corpus-wide tools are validated against known anchor cases (the BM.77056 cross-curricular pattern, the BM.47463 ↔ CBS.6060 Šurpu chain, and chronological diffusion of a top-host-count chunk). The 20-sample method is appropriate for per-tablet probes; corpus-wide enumeration is anchored differently.

## Audit script

`scripts/round5-corpus-wide-chunk-audit.mjs`. Five tests, each emits a final `✅ PASS` / `❌ FAIL` line; the script exits with status 2 if any test fails.

## Test 1 — Index build sanity

**Hypothesis:** `~/.cache/cuneiform-mcp/chunk-index.json` builds in under 15 minutes on a 35K-tablet corpus and the non-singleton hash count lands in the 100K-500K band expected for length-20 windows over post-pruning.

**Ground truth:** the build script writes `total_non_singleton_hashes`; the audit verifies the band.

**Secondary check:** sample 10 entries and reconstruct their signs from a fresh trigram pass over `all-signs-full.json`. Round-trip equality verifies the build script's `reconstructChunkSigns` matches the runtime expectation.

**Expected result:** PASS — both checks succeed.

## Test 2 — `find_formulaic_passages` positive control (BM.77056 cross-curricular)

**Hypothesis:** the BM.77056 position-57 cross-curricular chunks from v0.19's §3.9.1 follow-up surface in the top-50 at default `min_hosts=20`.

**Ground truth:** v0.19 documented three chunks at BM.77056 source position 57 (lengths 25, 22, 21) whose host sets collectively span six *āšipūtu* sub-genres. The chunk-hash index should pick these up by exact match; the genre-diversity weighting should rank them highly.

**Expected result:** PASS — at least one chunk containing BM.77056 surfaces in the top-50.

## Test 3 — `find_formulaic_passages` negative (colophon suppression)

**Hypothesis:** `Asb.c` / `Asb.d` colophon templates do not dominate the top-10 by `novelty_score`. These chunks have high host counts but their hosts are all Library of Ashurbanipal — `host_genres_spanned` collapses to one or two, so the genre-diversity multiplier penalizes them.

**Pass criterion:** at most 2 of the top-10 entries are purely Asb.* hosts. Stricter criteria are possible but require an enrichment burst to have populated genre attribution for the Asb.* prefixes; the ≤2/10 bar is sustainable across metadata-coverage regimes.

**Expected result:** PASS in normal-coverage regimes; FAIL would indicate the novelty weighting is wrong.

## Test 4 — `trace_chunk_diffusion` positive

**Hypothesis:** the highest-host-count chunk from Test 2 spans ≥2 distinct periods when traced through `traceChunkDiffusion`.

**Ground truth:** any canonical KAR-44 incipit chunk should diffuse Old Babylonian through Hellenistic in the documented transmission history of the *āšipūtu* curriculum. The conservative test bar is ≥2 periods.

**Expected result:** PASS.

## Test 5 — `build_citation_graph` positive (BM.47463 → CBS.6060)

**Hypothesis:** the BM.47463 ↔ CBS.6060 Šurpu commentary/base pair (documented in methods-paper §3.7.1 as a 147-sign shared chain) appears as an edge in `build_citation_graph` at `min_shared_chunks=1`.

**Ground truth:** v0.18.19's `commentary_quotes_base_text` verdict already confirmed this pair at pair-level granularity. The corpus-level graph must re-derive the same conclusion from the chunk-hash index alone.

**Pass criterion:** an edge between BM.47463 and CBS.6060 (in either direction; the graph orientation depends on which is tagged commentary genre in the metadata).

**Expected result:** PASS in normal-coverage regimes. FAIL would indicate either (a) genre metadata is missing for one or both tablets, (b) the chunk-hash index was built against a corpus that excludes one, or (c) the commentary/base partition is broken.

## Provenance of anchor choices

- **BM.77056** — v0.19 follow-up (commit `b913d72`), 13 chunks across 12 distinct *āšipūtu* sub-genres at post-enrichment.
- **BM.47463 ↔ CBS.6060** — v0.18.19 `commentary_quotes_base_text` verdict; methods-paper §3.7.1.
- **Asb.\*** — colophon prefix established in v0.18.19 calibration as the dominant precision-noise source for find_embedded_fragments; same role here.

## Sequencing dependency

This audit assumes:
1. `scripts/build-chunk-index.mjs` has run and the cache file exists.
2. `fragment_metadata_coverage` ≥10% — otherwise Tests 2, 3, 5 underrun their precision targets.
3. The v0.19.1 `host_genres_spanned` field is present on `find_chunk_parallels` outputs (Test 2 indirectly depends on this via `find_formulaic_passages`' ranking).

If the coverage gate is not met, the failure mode is graceful: the audit emits FAIL lines with diagnostic context (metadata coverage %, missing genre attributions) so the operator knows whether to extend the enrichment burst before re-running.
