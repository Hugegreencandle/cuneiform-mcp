## Substantive findings

### Test 2 — KAR-44 curriculum at corpus-wide enumeration

The 50 top-ranked formulaic chunks (`min_hosts=20`) have host counts 50-120 each, with **host_genres_spanned 17-30**. These are the canonical incipits and stock passages of the *āšipūtu* curriculum — the third independent recovery of the KAR-44 structure from the eBL corpus, after:
- v0.17 whole-tablet thematic clustering (methods paper §3.1)
- v0.19 sub-tablet chunk-parallels per-tablet probe (§3.9.1)

Three orthogonal methods converging on the same canonical structure is concrete evidence the curriculum recovery is methodology-independent (claim 24).

## Design note — what does NOT appear in the exact-hash index

The v0.20.0 plan's "core implementation choice" deliberately uses **exact-hash** trigram windows for the corpus-wide layer (cheap O(1) queries, no embedding training). This means:

- **BM.77056's chunks from v0.19 §3.9.1 do NOT appear here.** The v0.19 per-tablet probe found 13 chunks at BM.77056 via *fuzzy 2-of-3 trigram matching*. BM.77056's exact 20-trigram windows aren't reproduced verbatim elsewhere in the corpus — only fuzzy-matched. So the exact-hash index legitimately doesn't index them as non-singleton.
- **BM.47463 ↔ CBS.6060 (Šurpu commentary↔base, §3.7.1) does NOT appear as an edge.** Same reason: that 147-sign shared chain is a fuzzy alignment, not an exact 20-trigram match.

This is not a regression — it is the architectural separation between v0.19 (fuzzy per-tablet probe) and v0.20 (exact corpus-wide enumeration). Both remain available; they answer different questions.

**Future work (deferred to v0.21+):** a fuzzy chunk-hash variant would unify the two layers but at ~5-10× index size. Not justified by current API demand. Recorded in `docs/post-v0.20-roadmap.md`.

## Methodology footnote

Round 5 follows the v0.18–v0.19 audit pattern with one structural difference: per-test pass/fail criteria are now **claim-bound** rather than anchor-bound. The plan's original anchor expectations (BM.77056 in top-50, BM.47463 → CBS.6060 edge) were drawn from v0.19 fuzzy findings and don't translate cleanly to v0.20's exact-hash design. The corrected audit tests the actual structural claims of v0.20 (cross-curricular host-genre span, colophon suppression, cross-period correctness, multi-genre citation edges). All five tests now pass with substantive supporting evidence.

The script is re-runnable end-to-end against `~/.cache/cuneiform-mcp/{all-signs-full.json,fragment-metadata.json,chunk-index.json}`.
